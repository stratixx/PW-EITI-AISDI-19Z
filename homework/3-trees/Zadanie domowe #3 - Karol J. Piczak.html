Zadanie domowe #3

Do oceny mona zgosi maksymalnie trzy wybrane zadania domowe.

Materiay pomocnicze do zadania znajduj si w repozytorium na GitHub.

Nazwa repozytorium z rozwizaniem: AISDI-HW 3: Nazwisko

Tre zadania

Rozpatrujemy standardowe binarne drzewo poszukiwa, drzewo typu splay i drzewo AVL. Prosz naszkicowa kolejne stany poszczeglnych drzew po kadej z wykonywanych operacji:

  • wstaw kolejno wartoci 51, 22, 43, 84, 35, 86, 57, 98,

  • nastpnie znajd warto 57,

  • nastpnie usu kolejno wartoci 57, 51.

Schemat wywoania tych operacji wygldaby nastpujco:

trees.cpp
#include <vector>
template<typename Key>
struct Node {
    Key key;
    // ...
};
template<typename Key>
class Tree {
    public:
        // ...
        virtual void erase(const Key& key) { /* ... */ }
        virtual Node<Key>* insert(const Key& key) { /* ... */ }
        virtual Node<Key>* find(const Key& key) const { /* ... */ }
};
template<typename Key>
class BinarySearchTree : public Tree<Key> {
    // ...
};
template<typename Key>
class SplayTree : public Tree<Key> {        
    // ...
};
template<typename Key>
class AVLTree : public Tree<Key> {        
    // ...
};
// ...
    auto bst = BinarySearchTree<int>();
    auto splay = SplayTree<int>();
    auto avl = AVLTree<int>();
    std::vector<Tree<int>*> trees{};
    trees.push_back(&bst);
    trees.push_back(&splay);
    trees.push_back(&avl);
    // Operacje wykonujemy dla kadego typu drzewa
    for (const auto& tree : trees) {
        tree->insert(51); // ...-00.pdf
        tree->insert(22); // ...-01.pdf
        tree->insert(43); // ...-02.pdf
        tree->insert(84); // ...-03.pdf
        tree->insert(35); // ...-04.pdf
        tree->insert(86); // ...-05.pdf
        tree->insert(57); // ...-06.pdf
        tree->insert(98); // ...-07.pdf
        
        tree->find(57);   // ...-08.pdf
        
        tree->erase(57);  // ...-09.pdf
        tree->erase(51);  // ...-10.pdf
    }
// ...

Do realizacji szkicw mona wykorzysta na przykad pakiet Graphviz lub LaTeX.

W ramach niniejszego zadania domowego nie jest wymagana implementacja poszczeglnych drzew. Operacje mona "policzy" na kartce i na tej podstawie bezporednio przygotowa szkice.

Pomocne narzdzia

  • Na stronie viz-js.com mona poeksperymentowa z zapisem w jzyku DOT bezporednio w przegldarce, bez koniecznoci instalacji pakietu Graphviz.

  • VisuAlgo zawiera interaktywne wizualizacje drzewa BST, AVL i wielu innych struktur danych i algorytmw.

  • W repozytorium przygotowaem skrypt w Pythonie, ktry pozwala przyspieszy generowanie wykresw za pomoc XeLaTeX-u.

Forma dostarczenia wynikw

Prosz wgra wynikowe pliki do repozytorium oznaczajc je jednoznacznie co do kolejnoci (np. bst-00.png, ..., bst-10.png) lub scalajc w jeden obraz/PDF. Nie jest konieczne zaczanie plikw pomocniczych czy implementacji drzew. Do wizualizacji mona wykorzysta rwnie inne dowolne narzdzia. W ostatecznoci moe nimi by kartka i owek, ale preferowane s wersje elektroniczne.
